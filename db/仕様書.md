# DB仕様書

## 保存するjson

### ファイル構成


```
db
 L entry            記事関連のデータを保管する場所
    L index         バージョン管理等、記事の情報を記事IDごとに管理する
       L 0.json     ページIDでファイル名をつけるので、０からの連番
    L body          全記事、全バージョンのテキストの保管場所
       L 0_0.json   [記事ID]_[バージョン].json
       L 0_1.json   バージョンは０からの連番
       L 0_2.json
```
